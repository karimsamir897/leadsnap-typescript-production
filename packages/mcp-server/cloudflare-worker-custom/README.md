# thrive-mcp-custom-worker

A Cloudflare Worker that serves the Thrive/LeadSnap **MCP server** over HTTP, with OAuth-based session
auth and two working tools:

- **`search_docs`** — searches the SDK documentation (methods, params, examples) using a local
  in-memory index (`LocalDocsSearch`). No network call, no token needed.
- **`execute`** — runs user TypeScript against the Thrive SDK. Because Deno isn't available inside
  Workers, the code is delegated to a **remote Deno sandbox** and the result is streamed back.

```
MCP client (Claude.ai / Claude Desktop / Cursor / Inspector)
  │  OAuth: paste bearer token + pick environment on the consent screen
  ▼
this Worker  ──  search_docs ─▶ local in-memory index
             └─  execute     ─▶ POST  https://ls-code-sandbox-1.leadsnap.deno.net  (Deno sandbox)
```

> This package is hand-maintained and **not** generated by stlc, so it survives SDK regeneration
> (unlike the sibling `../cloudflare-worker`, which is overwritten and is currently broken).

---

## Prerequisites

- **Node.js 18+** (this repo uses 22).
- A **Cloudflare account** — only needed to deploy (`wrangler dev` runs fully locally).
- A **Thrive/LeadSnap bearer token** to authenticate when connecting an MCP client.

All tooling (`wrangler`, etc.) is installed locally via `npm install` — no global installs required.

## 1. Install

```bash
cd packages/mcp-server/cloudflare-worker-custom
npm install
```

Confirm the remote sandbox is reachable (should print `ok`):

```bash
curl https://ls-code-sandbox-1.leadsnap.deno.net
```

## 2. Configure

Configuration lives in [`wrangler.jsonc`](./wrangler.jsonc).

### a) KV namespace (required for OAuth)

The OAuth provider stores grants/tokens in KV. Create a namespace and paste the printed id into
`wrangler.jsonc` → `kv_namespaces[0].id` (it currently holds a placeholder):

```bash
npx wrangler kv namespace create OAUTH_KV
# -> copy the "id" value into wrangler.jsonc
```

> For **local dev only**, the placeholder id is fine — `wrangler dev` simulates KV locally. A real id
> is required before `deploy`.

### b) Sandbox URL (already set)

```jsonc
// wrangler.jsonc
"vars": { "CODE_SANDBOX_URL": "https://ls-code-sandbox-1.leadsnap.deno.net" }
```

### c) Sandbox shared secret (optional)

Only needed if the Deno sandbox enforces one (it currently does not):

```bash
# production:
npx wrangler secret put SANDBOX_SHARED_SECRET

# local dev: create a .dev.vars file (git-ignored) instead:
echo 'SANDBOX_SHARED_SECRET=your-secret' > .dev.vars
```

## 3. Run locally

```bash
npm run dev          # serves on http://localhost:8787
```

Open `http://localhost:8787/` in a browser — you should see the server home page.

---

## 4. Connect an MCP client

The MCP endpoints are `/mcp` (streaming HTTP) and `/sse` (legacy). Both require completing the OAuth
consent flow once: paste your **bearer token** and select an **environment** (Leadsnap / Thrive /
Staging / Local).

### MCP Inspector (quickest way to test)

```bash
npx @modelcontextprotocol/inspector
```

Set the server URL to `http://localhost:8787/mcp`, connect, and complete the consent screen.

### Claude Desktop / Cursor / Windsurf

Point the client at the worker URL using `mcp-remote` (see the in-app instructions rendered at
`http://localhost:8787/`), e.g.:

```json
{
  "mcpServers": {
    "thrive_mcp": {
      "command": "npx",
      "args": ["-y", "mcp-remote@latest", "http://localhost:8787/mcp"]
    }
  }
}
```

For production, replace the URL with your deployed `https://…workers.dev/mcp`.

---

## 5. Test the tools

After connecting:

- **List tools** → you should see `search_docs` and `execute`.
- **search_docs**
  ```json
  { "query": "heatmap grid points", "language": "typescript" }
  ```
  Returns indexed SDK methods/examples (proves local search is initialized).
- **execute** (no Thrive call)
  ```json
  { "code": "async function run(client){ return 1 + 1; }" }
  ```
  Returns `2`.
- **execute** (real Thrive call — needs a valid token + the `Thrive` environment)
  ```json
  { "code": "async function run(client){ return await client.public.api.v1.heatmap.generateGridPoints({ grid_radius: 500, grid_size: 7, lat: 37.7749, lng: -122.4194 }); }" }
  ```

## 6. Deploy

```bash
npm run deploy
```

After deploying:

1. Make sure `kv_namespaces[0].id` in `wrangler.jsonc` is a **real** id (step 2a).
2. If you use a shared secret, set it on the deployed worker: `npx wrangler secret put SANDBOX_SHARED_SECRET`.
3. Update your MCP client(s) to the deployed URL: `https://thrive-mcp-custom-worker.<your-subdomain>.workers.dev/mcp`.

## Configuration reference

| Name | Where | Required | Purpose |
|---|---|---|---|
| `CODE_SANDBOX_URL` | `wrangler.jsonc` → `vars` | yes | Remote Deno sandbox URL for the `execute` tool. |
| `OAUTH_KV` | `wrangler.jsonc` → `kv_namespaces` | yes | KV namespace backing OAuth tokens/grants. |
| `MCP_OBJECT` | `wrangler.jsonc` → `durable_objects` | yes | Durable Object hosting each MCP session. |
| `SANDBOX_SHARED_SECRET` | secret / `.dev.vars` | no | Forwarded as the `Authorization` header to the sandbox. |

**Session inputs** (collected per-connection on the OAuth consent screen, not configured here):
the **bearer token** (forwarded to the sandbox as `THRIVE_MCP_BEARER_TOKEN`) and the **environment**
(forwarded as `client_opts.environment`).

---

## How it works

- Auth/session: `@cloudflare/workers-oauth-provider` + the `agents` `McpAgent` (a Durable Object).
  The consent form persists `{ bearerToken, environment }` for the session.
- `search_docs`: on session init, `setLocalSearch(await LocalDocsSearch.create())` builds an in-memory
  index from the SDK's embedded methods + README (no `docsDir`, so no filesystem access — Worker-safe).
- `execute`: `POST`s to `CODE_SANDBOX_URL` with `x-stainless-mcp-client-envs:
  {"THRIVE_MCP_BEARER_TOKEN": "…"}`, body `{ project_name, code, intent?, client_opts: { environment } }`,
  and optional `Authorization: <SANDBOX_SHARED_SECRET>`; it maps the sandbox's
  `{ is_error, result, log_lines, err_lines }` back into MCP content blocks.

## Troubleshooting

- **`search_docs` returns "Local search not initialized"** — should not happen here; this worker calls
  `setLocalSearch()` in `init()`. (This is exactly the bug in the generated `../cloudflare-worker`.)
- **401 on `/mcp` or `/sse`** — expected until you complete the OAuth consent. Clear stale auth with
  `rm -rf ~/.mcp-auth` and reconnect.
- **`execute` returns "Missing THRIVE_MCP_BEARER_TOKEN…"** — the connection has no bearer token;
  re-authenticate and ensure you pasted a token on the consent screen.
- **`execute` hits the wrong API host** — the sandbox resolves the base URL from `environment`. Pick
  `Thrive` for `app.crmthrive.com`. If the sandbox's SDK doesn't recognize the env name, forward
  `THRIVE_MCP_BASE_URL` in `src/execute-tool.ts` instead (computed from the environment).
- **`deploy` fails on KV** — `kv_namespaces[0].id` is still the placeholder; create a real namespace (step 2a).
- **Weird `instanceof` / duplicate SDK errors** — keep `@modelcontextprotocol/sdk` pinned to `1.28.0`
  everywhere (see the `overrides`/`resolutions`/`pnpm.overrides` blocks in `package.json`); `agents`
  requires that exact version.
